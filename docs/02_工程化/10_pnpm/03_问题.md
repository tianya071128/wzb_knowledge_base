# 问题(FAQ)

## 每个成员的 `pnpm` 版本不同, 如何在一个项目中统一版本？

使用 `corepack` 解决, **是 Node.js 官方自带的「包管理器管理器」**

作用只有一个，**统一管理 npm /yarn/pnpm 的版本，强制项目使用固定的包管理器版本，不随系统变化。**

它是 Node.js **从 v14.19.0 /v16.9.0 开始内置**的工具，**不需要单独安装**。

步骤:

1. 启用 `corepack`

   ```bash
   corepack enable
   ```

2. 在项目 `package.json` 中定义：

    ```json
    "packageManager": "pnpm@9.11.0" --> 指定版本
    ```

3. 此时在项目根目录下执行

   ```bash
   # 9.11.0
   pnpm -v
   
   # 直接安装，使用指定版本
   pnpm install
   ```

::: tip 注意

如果成员没有启用 `corepack`, 安装依赖还是会按照全局的版本安装。

此时，可以使用在 `package.json` 加上  `preinstall` 钩子进行检测

:::

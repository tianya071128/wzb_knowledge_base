# 可选选项

RFC 793 设计 TCP 头部时，特意预留了最多 40 字节的选项空间，让 TCP 可以在不改变头部整体结构的情况下，不断增加新功能。所有现代 TCP 的性能提升（高速传输、高效重传、精确计时），几乎都依赖于这些选项。

## 整体结构

TCP 可选选项位于**固定头部之后、数据载荷之前**，长度可变，最多 40 字节。

::: tip 为什么最多 40 字节？

因为 TCP 头部的「数据偏移」字段是 4 位，最大值 15，头部总长度最大 15×4=60 字节，减去固定头部 20 字节，正好剩下 40 字节

:::

### 通用 TLV 格式

大部分选项都遵循统一的 **TLV（Type-Length-Value）** 格式：

```plaintext
类型(1字节) + 长度(1字节) + 数据(可变长度)
```

- **类型（Type）**：唯一标识选项类型，由 IANA 统一分配，目前已分配超过 30 种
- **长度（Length）**：整个选项的总长度（包括类型和长度字段本身）
- **数据（Value）**：选项的具体内容，长度由长度字段决定

### 特殊格式

有两个选项不遵循 TLV 格式，只有 1 字节的类型字段，没有长度和数据：

| 选项                | 类型值 | 作用                                                         |
| ------------------- | ------ | ------------------------------------------------------------ |
| **EOL（选项结束）** | 0      | 标记选项部分的结束，后面的所有字节都被视为填充，接收方必须忽略 |
| **NOP（无操作）**   | 1      | 用于填充，使选项总长度对齐到 4 字节的整数倍（因为数据偏移的单位是 4 字节） |

**关键规则**：

- EOL 只能出现在选项部分的最后
- NOP 可以出现在任何位置，接收方必须忽略
- 选项总长度必须是 4 字节的整数倍，不足时用 NOP 填充

## 选项

### MSS（Maximum Segment Size，最大段大小）

**MSS 是 TCP 性能的基石参数**，它直接决定了单个 TCP 报文能携带多少数据，是避免 IP 分片、最大化网络吞吐量的核心机制。

#### 基本属性

- **选项类型**：2（TCP 可选选项编号）

- **选项长度**：4 字节（固定）

- **数据格式**：2 字节无符号整数，取值范围 0~65535

- **生效时机**：**只能在 SYN 和 SYN+ACK 报文中出现**，连接建立后永久不能改变

- **本质含义**：**接收方能够接收的单个 TCP 报文的最大数据载荷长度**

- **MSS 只计算 TCP 数据载荷的长度，不包括任何头部**

- **标准计算公式**:

  ```plaintext
  MSS = 路径 MTU - IP 头部最小长度(20字节) - TCP 头部最小长度(20字节)
  ```

#### MSS 的作用

MSS 的唯一设计目的就是**彻底避免 IP 分片**。

IP 分片有一个无法解决的痛点：**只要有一个分片丢失，整个原始 IP 数据报就全部作废，TCP 必须重传整个 TCP 段**。

### 窗口缩放（Window Scale）

#### 基本属性

* **选项类型**：3（TCP 可选选项编号）
* **选项长度**：3 字节（固定）
* **数据格式**：1 字节无符号整数，表示**缩放因子（Shift Count）**
* **取值范围**：`0 ~ 14`（RFC 1323 强制规定，超过 14 按 14 处理）
* **生效时机**：**只能在 SYN 和 SYN+ACK 报文中协商**，连接建立后永久不能改变
* **本质作用**：将 16 位窗口大小字段的表示范围从 64KB 扩展到最大 **1GB**
* **计算公式**: `实际窗口大小 = TCP 头部窗口大小字段值 × 2^缩放因子`

#### 作用

窗口缩放的唯一设计目的就是**解决带宽延迟乘积（BDP）瓶颈**。

::: tip 什么是带宽延迟乘积？

带宽延迟乘积（Bandwidth-Delay Product, BDP）是指**在数据从发送方传输到接收方的往返时间内，网络中能够容纳的最大数据量**。

```plaintext
BDP = 链路带宽 × 往返延迟（RTT）
```

:::

### SACK 允许(SACK-Permitted)

#### 基本属性

* **选项类型**：5（TCP 可选选项编号）
* 长度可变，**每 1 个 SACK 区间占 8 字节**
* 一个 TCP 报文最多放**4 个 SACK 块**

```plaintext
1字节 Kind(5)
1字节 Length
4字节 左边界序号 (Left Edge)
4字节 右边界序号 (Right Edge)
...其他块
```

#### 作用

1. 解决**累积确认只能认连续流**的缺陷
2. 发送方只重传**空缺段**，不重传已到达数据
3. [搭配快速重传，大幅减少无效重传流量](/tcp/data_retransmission.html#sack-选择性确认)

## 选项的协商

**SYN 报文协商**：所有需要双方同意的选项（MSS、窗口缩放、SACK 允许、时间戳、ECN 允许、快速打开）都必须在 SYN 报文中发送

**单向协商**：选项是单向的，客户端发送的选项只适用于客户端到服务端的方向，服务端需要在 SYN+ACK 报文中回复自己支持的选项

**忽略未知选项**：接收方必须忽略所有未知类型的选项，不能因为未知选项而丢弃报文

## 注意事项

1. **大部分重要选项（MSS、窗口缩放、SACK 允许）只能在三次握手阶段协商，连接建立后不能改变**
2. **选项的顺序不影响功能，接收方必须能够处理任何顺序的选项**

# 建立连接

**三次握手是 TCP 可靠传输的起点**，它的唯一核心目的是**同步双方的初始序列号（ISN），并确认双方都具备收发数据的能力**

## TCP 连接是什么?

**TCP 连接是纯逻辑的，不是物理的**。它只是**两端操作系统内核中，各自维护的一个数据结构**，这个数据结构记录了和这个通信会话相关的所有状态信息：

- 四元组（源 IP: 源端口 + 目的 IP: 目的端口）
- 双方的初始序列号（ISN）
- 双方的当前发送序列号和接收确认号
- 双方的窗口大小和缩放因子
- MSS（最大段大小）
- 连接状态（LISTEN、SYN_SENT、ESTABLISHED 等）
- 各种计时器（重传计时器、保活计时器等）
- 发送缓冲区和接收缓冲区

**三次握手的过程，就是双方交换这些状态信息，并在各自内核中创建这个数据结构的过程**。

## 为什么必须建立连接？

核心理由：**实现可靠的字节流传输**

1. **同步双方的初始序列号（ISN）—— 可靠传输的基石**

   TCP 给传输的**每一个字节**都分配一个唯一的序列号，所有的确认应答、超时重传、有序性保证，全部建立在序列号之上。**三次握手的本质，就是让双方互相确认对方的初始序列号**

2. **协商所有 TCP 扩展选项 —— 现代 TCP 性能的基础**

   很多 TCP 选项（MSS、窗口缩放、SACK、时间戳、ECN 等），**都必须在连接建立阶段协商完成**，连接建立后不能改变

3. **为通信会话分配内核资源**

4. **验证双方的收发能力**

   三次握手可以同时验证客户端和服务端的发送能力和接收能力都正常

5. **避免旧连接的残留数据包干扰**

   互联网是不可靠的，数据包可能在网络中延迟很久才到达。这些残留的旧连接数据包可能会被新的通信误认为是有效的，导致严重的数据错误。

   三次握手通过随机生成初始序列号（ISN），完美解决了这个问题。新连接的 ISN 和旧连接的 ISN 几乎不可能相同，所以残留的旧数据包的序列号会落在新连接的接收窗口之外，被直接丢弃。

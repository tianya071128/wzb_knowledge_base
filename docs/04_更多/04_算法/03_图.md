# 图(graph)

图（graph）是一种非线性数据结构，由顶点（vertex）和边（edge）组成。其核心特点是多对多关联，能高效模拟现实世界中的网络结构。

## 图的关键术语

| <div style="width: 100px">分类</div>               | <div style="width: 60px">术语</div>       | 英文表示                           | 简要说明                                                     |
| ------------------ | ---------- | ---------------------------------- | ------------------------------------------------------------ |
| **基础构成**       | 顶点       | Vertex（复数：Vertices）           | 图中的核心实体，也常被称为 “节点（Node）”，是关系的承载者。  |
|                    | 边         | Edge                               | 连接两个顶点的线，用于表示顶点间的关系。                     |
|                    | 图         | Graph                              | 形式化定义为 \(G=(V,E)\)，其中 V 是顶点集合，E 是边集合。    |
| **图的核心类型**   | 无向图     | Undirected Graph                   | 边无方向，边 \((u,v)\) 与 \((v,u)\) 等价，如社交网络的朋友关系。 |
|                    | 有向图     | Directed Graph（简称 Digraph）     | 边有方向，边 \(⟨u,v⟩) 表示从顶点 u 指向顶点 v，如网页链接。  |
|                    | 带权图     | Weighted Graph                     | 边附带数值属性（权重），也称为 “网（Network）”，如地图中带距离的道路。 |
|                    | 有向无环图 | Directed Acyclic Graph（简称 DAG） | 无环的有向图，是拓扑排序的唯一适用场景。                     |
|                    | 稠密图     | Dense Graph                        | 边的数量接近顶点数的平方（\(n^2\)），适合用邻接矩阵存储。    |
|                    | 稀疏图     | Sparse Graph                       | 边的数量远小于 \(n^2\)，适合用邻接表存储。                   |
| **顶点与边的属性** | 度         | Degree                             | 无向图中，顶点连接的边的数量。                               |
|                    | 入度       | In-degree                          | 有向图中，指向当前顶点的边的数量。                           |
|                    | 出度       | Out-degree                         | 有向图中，从当前顶点出发的边的数量。                         |
|                    | 权重       | Weight/Cost                        | 边或顶点附带的数值，用于表示距离、成本、优先级等。           |
|                    | 自环       | Self-loop                          | 起点和终点为同一个顶点的边，如社交网络中用户给自己点赞的记录。 |
|                    | 平行边     | Parallel Edges                     | 两个相同顶点间存在多条无向边，或两个相同方向顶点间存在多条有向边。 |
| **路径与环相关**   | 路径       | Path                               | 由一系列顶点和边组成的序列，顶点可重复（简单路径中顶点不重复）。 |
|                    | 环         | Cycle                              | 起点和终点为同一个顶点，且除起点外其他顶点不重复的路径；有向图中的环称为 “有向环（Directed Cycle）”。 |
|                    | 路径长度   | Path Length                        | 路径中包含的边的数量，带权图中可指边的权重总和。             |
| **连通性相关**     | 连通图     | Connected Graph                    | 无向图中，任意两个顶点之间都存在可达路径。                   |
|                    | 强连通图   | Strongly Connected Graph           | 有向图中，任意两个顶点 u 和 v，既存在 u 到 v 的路径，也存在 v 到 u 的路径。 |
|                    | 弱连通图   | Weakly Connected Graph             | 有向图忽略边的方向后成为连通图，但本身不是强连通图。         |
|                    | 连通分量   | Connected Component                | 无向图中最大的连通子图；有向图中对应 “强连通分量（Strongly Connected Component，简称 SCC）”。 |
| **子图与特殊子图** | 子图       | Subgraph                           | 由原图的部分顶点和部分边组成的图（顶点和边均属于原图）。     |
|                    | 生成树     | Spanning Tree                      | 包含原图所有顶点，且边数为 \(n-1\)（n 为顶点数）的无环子图。 |
|                    | 最小生成树 | Minimum Spanning Tree（简称 MST）  | 带权图中，所有生成树里边的权重总和最小的生成树。             |
|                    | 最短路径   | Shortest Path                      | 带权图中，两个顶点间权重总和最小的路径。                     |
| **存储与算法相关** | 邻接矩阵   | Adjacency Matrix                   | 用二维数组存储图的邻接关系，适合稠密图。                     |
|                    | 邻接表     | Adjacency List                     | 用数组 + 链表（或数组）存储图，数组存顶点，链表存邻接点，适合稀疏图。 |
|                    | 拓扑排序   | Topological Sorting                | 对 DAG 的顶点排序，使所有有向边的起点在终点之前。            |
|                    | 并查集     | Disjoint Set Union（简称 DSU）     | 常用于图的连通性判断、Kruskal 算法等，高效管理和合并顶点集合。 |

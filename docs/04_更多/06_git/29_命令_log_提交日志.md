# git log

## 作用

用于**查看提交历史**的核心命令，它能展示仓库的提交记录、分支轨迹、代码变更等信息

## 概述

```bash
# 路径需要再前面添加 -- 分隔
git log [选项] [修订范围] [-- [路径]]
```

`[选项]`：控制输出格式、范围、内容的参数（核心部分）

`[修订范围]`：可选，指定要查看的提交范围

`[路径]`：可选，指定文件 / 目录，仅查看该路径的提交历史

::: tip 注意

**顺序问题**：`git log` 的参数顺序为「选项 → 修订范围 → 路径」，路径必须放在最后，用 `--` 分隔；

**路径通配符**：使用通配符（如 `*.js`）时，建议用引号包裹，避免被 shell 提前解析；

:::

## 修订范围: 指定要查看的提交集合

修订范围用于告诉 Git 「我要看哪段时间 / 哪些分支 / 哪几个提交的历史」，以下是最常用的形式：

### 单个提交 / 引用：查看从该提交往前的所有历史

直接指定一个**提交哈希**、**分支名**、**标签名**或**相对引用**（如 `HEAD~3`），Git 会显示从该提交开始，一直追溯到仓库初始提交的所有历史。

```bash
# 查看从提交 a1b2c3d 往前的所有历史（包含 a1b2c3d）
git log a1b2c3d

# 查看 feature 分支的所有历史
git log feature

# 查看标签 v1.0.0 指向的提交往前的所有历史
git log v1.0.0

# 查看从当前提交往前数第 3 个提交的历史（包含该提交）
git log HEAD~3
```

### 双点范围（<提交1>..<提交2>）：查看两个提交之间的历史

- `A..B`: 从 `A` 之后到 `B`, 不包含 `A`的提交
  - 若省略 `<提交1>`，默认使用 `HEAD`（即 `..B` 等同于 `HEAD..B`）；
  - 若省略 `<提交2>`，默认使用 `HEAD`（即 `A..` 等同于 `A..HEAD`）。
- `A^..B`: 从 `A` 之后到 `B`, 包含 `A` 的提交

```bash
# 查看从 a1b2c3d 之后到 d4e5f6g 的所有提交（不包含 a1b2c3d，包含 d4e5f6g）
git log a1b2c3d..d4e5f6g

# 查看从 feature 分支分叉后，main 分支新增的所有提交
git log feature..main

# 查看从 a1b2c3d 到 d4e5f6g 的所有提交（包含 a1b2c3d 和 d4e5f6g）
git log a1b2c3d^..d4e5f6g
```

### 三点范围（<提交1>...<提交2>）：查看两个分支的「独有提交」

- `A...B`: 只显示 `A` 和 `B` 各自独有的提交，不显示它们的共同祖先. **特别适合对比两个分支的差异**
  - 显示所有**能从 A 或 B 到达，但不能从它们的共同祖先到达**的提交；
  - 配合 `--left-right` 选项使用，能直观看到哪些提交属于 A，哪些属于 B。

```bash
# 查看 main 和 feature 分支各自独有的提交（不显示共同祖先）
git log --oneline --left-right main...feature
```

### 多引用组合：同时查看多个分支 / 标签的历史

可以直接列出多个分支名、标签名或提交哈希，Git 会显示所有能从这些引用到达的提交。

```bash
# 同时查看 main 和 feature 分支的历史
git log main feature

# 同时查看标签 v1.0.0 和 v1.1.0 的历史
git log v1.0.0 v1.1.0
```

### 负引用：排除特定提交的历史

在引用前加 `^` 或 `--not`，可以排除能从该引用到达的所有提交。

```bash
# 查看 main 分支的历史，但排除 feature 分支的提交
git log main ^feature
# 等同于
git log main --not feature
```

## 路径: 筛选只修改了特定文件 / 目录的提交

路径用于告诉 Git 「只看修改了这些文件 / 目录的提交」，会自动过滤掉未修改指定路径的提交。路径需放在命令最后，用 `--` 分隔（避免路径被识别为选项）。

### 基本用法：指定单个 / 多个文件或目录

- 路径支持通配符（如 `\*.js` 表示所有 `JS` 文件），但需注意转义（避免被 `shell` 提前解析）

```bash
# 只查看修改了 src/login.js 的提交
git log -- src/login.js

# 只查看修改了 src/ 目录的提交
git log -- src/

# 只查看修改了 src/login.js 和 src/pay.js 的提交
git log -- src/login.js src/pay.js

# 查看所有 JS 文件的提交历史（用引号包裹通配符）
git log -- '*.js'
```

### 跟随文件重命名：--follow

如果文件被重命名过，直接用路径筛选只能看到重命名后的历史；加上 `--follow` 选项，Git 会**跟随文件的重命名轨迹**，显示重命名前的完整历史。

- `--follow` 只能配合**单个文件**使用，不能用于目录；
- 它通过检测文件内容的相似度来识别重命名，若文件内容变化太大可能失效。

```bash
# 查看 src/login.js 的完整历史，包括它重命名前叫 src/old-login.js 时的历史
git log --follow -- src/login.js
```

### 显示文件变更统计：配合 --stat/--numstat

路径筛选可以和 `--stat`/`--numstat` 组合，显示指定路径的变更统计。

```bash
# 查看 src/login.js 的提交历史及每次的变更统计
git log --stat -- src/login.js
```

```plaintext
commit a1b2c3d...
Author: 张三...
Date: ...

    新增登录功能

 src/login.js | 15 +++++++++++++++
 1 file changed, 15 insertions(+)

commit d4e5f6g...
Author: 李四...
Date: ...

    修复登录校验

 src/login.js | 4 ++--
 1 file changed, 2 insertions(+), 2 deletions(-)
```

## 选项

### 格式控制类选项：定制提交历史的显示样式

这类选项用于调整提交信息的输出格式

#### `--oneline`: 单行显示

- **作用**: 将每个提交压缩成**单行显示**，包含「简写提交哈希（前 7 位） + 提交信息标题」
- **示例**: `git log --oneline`

#### `--pretty=<格式>`: 自定义输出格式

- **作用**: 完全自定义提交信息的输出格式，支持预设格式和自定义占位符，是格式控制的核心选项

- **预设格式取值**：

  | 预设值             | 作用                                                                |
  | ------------------ | ------------------------------------------------------------------- |
  | `oneline`          | 等同于 `--oneline`                                                  |
  | `short`            | 显示「哈希 + 作者 + 提交信息标题」                                  |
  | `medium`           | Git 默认格式（哈希 + 作者 + 日期 + 完整提交信息）                   |
  | `full`             | 显示「哈希 + 作者 + 提交者 + 完整提交信息」                         |
  | `fuller`           | 显示「哈希 + 作者 + 作者日期 + 提交者 + 提交者日期 + 完整提交信息」 |
  | `email`            | 以电子邮件格式显示（适合转发）                                      |
  | `raw`              | 显示 Git 内部原始格式（包含树对象、父提交哈希）                     |
  | `format:<字符串>`  | 自定义格式字符串（见下文占位符）                                    |
  | `tformat:<字符串>` | 类似 `format`，但会自动在最后添加换行符                             |

- **示例**:
  ```bash
  # 自定义格式：显示「简写哈希 | 作者名 | 相对时间 | 提交信息标题」
  git log --pretty=format:"%h | %an | %ar | %s"
  ```

#### `--date=<格式>`: 自定义日期格式

- **作用**: 配合 `--pretty` 使用，定制日期的显示格式。
- **示例**:
  ```bash
  # 显示 ISO 格式的日期
  git log --pretty=format:"%h | %ad | %s" --date=iso
  ```

#### `--abbrev-commit, --abbrev=<n>, --no-abbrev-commit`: 控制提交哈希的输出格式

- **作用**:
  - `--abbrev-commit`: 使用简写哈希（默认 7 位）
  - `--abbrev=<n>`: 指定简写哈希的位数（如 `--abbrev=12`）
  - `--no-abbrev-commit`: 显示完整哈希
- **示例**: `git log --abbrev=10`

#### `--abbrev-commit, --abbrev=<n>, --no-abbrev-commit`: 控制提交哈希的输出格式

- **作用**:
  - `--abbrev-commit`: 使用简写哈希（默认 7 位）
  - `--abbrev=<n>`: 指定简写哈希的位数（如 `--abbrev=12`）
  - `--no-abbrev-commit`: 显示完整哈希
- **示例**: `git log --abbrev=10`

### 范围筛选类选项：精准定位要查看的提交

这类选项用于从「时间、作者、提交信息」等维度筛选提交，只显示需要的部分。

#### `-n <数量>, -<数量>`: 限制显示的提交数量

- **作用**: 限制显示的提交数量，只显示最近的 N 个提交
- **示例**: `git log -5 / git log -n 5`

#### `--skip=<数量>`: 跳过指定数量的提交

- **作用**: 跳过指定数量的提交，然后显示剩余的提交（适合分页查看历史）
- **示例**:
  ```bash
  # 跳过最近 3 个提交，显示之后的 5 个提交
  git log --skip=3 -5
  ```

#### `--since=<时间>, --after=<时间>`: 指定时间之后的提交

- **作用**: 只显示指定时间之后的提交

- **支持的时间格式**:
  - 相对时间：`"2 days ago"`、`"1 week ago"`、`"3 months ago"`、`"yesterday"`、`"today"`；

  - 绝对时间：`"2026-03-01"`、`"2026-03-01 10:00"`、`"Mar 5 2026"`。

- **示例**:

  ```bash
  # 显示最近 2 天的提交
  git log --since="2 days ago"

  # 显示 2026 年 3 月 1 日之后的提交
  git log --after="2026-03-01"
  ```

#### `--until=<时间>, --before=<时间>`: 指定时间之前的提交

- **作用**: 只显示指定时间之前的提交，时间格式同 `--since`。

- **示例**:

  ```bash
  # 显示 1 周前的提交
  git log --until="1 week ago"

  # 显示 2026 年 3 月 1 日之前的提交
  git log --before="2026-03-01"
  ```

#### `--author=<模式>, --committer=<模式>`: 指定作者的提交

- **作用**: 只显示指定作者的提交，支持**模糊匹配**（匹配姓名或邮箱）。
- `--committer=<模式>`: 只显示指定提交者的提交(将代码提交到仓库的人，可能与 `Author` 不同)
- **示例**:

  ```bash
  # 显示张三的提交
  git log --author="张三"

  # 显示邮箱包含 example.com 的提交
  git log --author="example.com"
  ```

#### `--grep=<模式>`: 提交信息的筛选

- **作用**: 只显示提交信息（标题 + 正文）中包含指定模式的提交，支持正则表达式
- **示例**:
  - **支持多个 `--grep=<模式>`**
  - 存在多个时, 只要匹配到了一个模式, 就会显示该提交
- **示例**:

  ```bash
  # 显示提交信息中包含"登录"的提交
  git log --grep="登录"

  # 使用正则表达式：显示提交信息中包含"修复"且后面跟"bug"的提交
  git log --grep="修复.*bug"
  ```

#### `--regexp-ignore-case, -i`: 不区分大小写

- **作用**: 使 `--grep`、`--author`、`--committer` 的匹配**不区分大小写**
- **示例**:

  ```bash
  # 不区分大小写，匹配"login"、"Login"、"LOGIN"
  git log --grep="login" --i
  ```

#### `--all-match`: 匹配所有 `--grep` 模式

- **作用**: 要求提交信息匹配**所有** `--grep` 模式（默认是匹配任意一个）。
- **示例**:

  ```bash
  # 显示提交信息中同时包含"修复"和"登录"的提交
  git log --grep="修复" --grep="登录" --all-match
  ```

#### `--invert-grep`: 反向匹配 `--grep` 模式

- **作用**: 反向匹配，只显示**不包含** `--grep` 模式的提交。
- **示例**:

  ```bash
  # 显示提交信息中不包含"测试"的提交
  git log --grep="测试" --invert-grep
  ```

### 内容展示类选项：查看提交的代码变更与文件信息

这类选项用于展示提交的具体内容，比如代码变更、文件列表、变更统计等。

#### `--patch`: 显示具体代码变更

- **作用**: 显示每个提交的**具体代码变更（diff）**，是追溯代码修改的核心选项。
- **示例**:

  ```bash
  # 显示最近 2 个提交的代码变更
  git log -p -2
  ```

#### `--no-patch, -s`: 默认行为, 不显示具体代码变更

- **作用**: 默认行为, 只显示提交的元信息（作者、时间、提交信息），**完全忽略代码变更**（与 `-p` 相反）。

#### `--stat`: 显示提交的变更统计

- **作用**: 显示提交的变更统计（文件名、增删行数），但不显示具体代码变更。

- **示例**:

  ```bash
  # 显示最近 2 个提交的代码变更
  git log -stat -2
  ```

- **输出示例**

  ```plaintext
  commit a1b2c3d...
  Author: 张三...
  Date: ...

      新增登录功能，包含手机号校验

   src/login.js | 15 +++++++++++++++
   src/api.js   |  4 ++--
   2 files changed, 17 insertions(+), 2 deletions(-)
  ```

#### `--shortstat`: 只显示变更的文件数、总增删行数

- **作用**: 只显示变更的**文件数、总增删行数**，比 `--stat` 更精简。

#### `--dirstat`: 按目录显示变更统计

- **作用**: 按目录显示变更统计（百分比），适合快速了解哪些目录修改最多。

#### `--numstat`: 以数字格式显示变更统计

- **作用**: 以数字格式显示变更统计（新增行数、删除行数、文件名）

#### `--name-only`: 只显示提交变更的文件名列表

- **作用**: 只显示提交变更的文件名列表，不显示其他信息。

#### `--name-status`: 显示变更文件名 + 变更状态

- **作用**: 显示变更文件名 + 变更状态

### 图形化与分支类选项：直观展示分支轨迹与合并关系

这类选项用于可视化分支的分叉、合并历史，以及筛选合并提交。

#### `--graph`: 图形展示

- **作用**: 用 ASCII 字符绘制分支合并轨迹，直观展示分支的分叉和合并，是查看分支历史的核心选项。

- **示例**:

  ```bash
  git log --graph --oneline --decorate
  ```

- **输出示例**

  ```plaintext
  *   m7n8o9p (HEAD -> main) Merge branch 'feature' into main
  |\
  | * b3c4d5e (feature) 优化登录样式
  | * a1b2c3d 新增登录接口
  * | h8i9j0k 修复支付超时
  * | g7h8i9j 初始化项目
  |/
  * f6g7h8i 初始提交
  ```

#### `--simplify-by-decoration`: 只显示有标签或分支指向的提交

- **作用**: 只显示有标签或分支指向的提交，简化图形化输出（隐藏中间的普通提交）

#### `--merges`: 只显示合并提交

- **作用**: 只显示合并提交（有多个父提交的提交）

#### `--no-merges`: 排除合并提交

- **作用**: 只显示非合并提交（排除合并提交）。

#### `--left-right`: 配合三点范围（A...B） 使用

- **作用**: 配合三点范围（A...B） 使用，在提交前标记 <（属于左边的提交 A）或 >（属于右边的提交 B），直观区分两个分支的独有提交。

### 搜索与匹配类选项：查找特定的代码变更

这类选项用于搜索提交中**添加/删除/修改**了特定代码的提交，是追溯代码变更的利器。

#### `-S <字符串>`: 搜索提交中添加或删除了指定字符串的提交

- **作用**: 搜索提交中**添加或删除**了指定字符串的提交（即 “字符串的出现次数发生变化” 的提交）

- **示例**:

  ```bash
  # 搜索添加或删除了 validatePhone 字符串的提交
  git log -S "validatePhone" --oneline
  ```

#### `-G <正则表达式>`: 搜索提交中添加或删除了指定字符串的提交

- **作用**: 搜索提交中**变更内容匹配**指定正则表达式的提交（即 “diff 中包含匹配正则的行” 的提交）。

- **与 `-S` 的区别**：`-S` 关注 “字符串出现次数的变化”，`-G` 关注 “diff 中是否有匹配的行”（即使次数不变，只是修改了字符串周围的代码，`-G` 也能匹配到）。

- **示例**:

  ```bash
  # 搜索变更内容包含 function login 的提交
  git log -G "function login" --oneline
  ```

#### `-L <起始行>,<结束行>:<文件路径> / -L :<函数名>:<文件路径>`: 指定行范围或指定函数的变更历史

- **作用**: 只显示指定文件的**指定行范围**或**指定函数**的变更历史，是追溯特定代码块的核心选项。

- **两种用法**：
  1. **行范围**：`-L 1,10:src/login.js`（显示第 1 到 10 行的变更）；
  2. **函数名**：`-L :validatePhone:src/login.js`（显示 `validatePhone` 函数的变更，Git 会自动识别函数范围）。

- **示例**:

  ```bash
  # 显示 src/login.js 中 validatePhone 函数的变更历史
  git log -L :validatePhone:src/login.js
  ```

#### `--follow`: 跟随文件的重命名历史

- **作用**：配合**路径筛选**使用，**跟随文件的重命名历史**（直接用路径筛选只能看到重命名后的历史，`--follow` 会显示重命名前的完整历史）。

  **限制**：只能用于**单个文件**，不能用于目录。

- **示例**:

  ```bash
  # 查看 src/login.js 的完整历史，包括重命名前的历史
  git log --follow -- src/login.js
  ```

### 性能与辅助类选项：优化显示与特殊场景

这类选项用于优化性能、调整排序、处理特殊场景（如重命名检测）。

#### `--all`: 显示所有分支、标签、远程分支的提交历史

- **作用**: 显示**所有分支、标签、远程分支**的提交历史（默认只显示当前分支的历史）

- **示例**:

  ```bash
  git log --all --graph --oneline --decorate
  ```

#### `--branches[=<模式>]`: 只显示本地分支的提交历史

- **作用**: 只显示**本地分支**的提交历史，可指定模式（如 `feature/*` 表示只显示以 `feature/` 开头的本地分支）

- **示例**:

  ```bash
  # 只显示以 feature/ 开头的本地分支的历史
  git log --branches=feature/* --oneline
  ```

#### `--tags[=<模式>]`: 只显示标签的提交历史

- **作用**: 只显示**标签**的提交历史，用法同 `--branches`。

- **示例**:

  ```bash
  git log --tags --oneline
  ```

#### `--remotes[=<模式>]`: 只显示远程分支的提交历史

- **作用**：只显示**远程分支**的提交历史，用法同 `--branches`。

- **示例**:

  ```bash
  # 只显示 origin/ 开头的远程分支的历史
  git log --remotes=origin/* --oneline
  ```

# git show

## 作用

用于**查看单个 Git 对象详细信息**的核心命令，它能精准展示提交、分支、标签、暂存区等对象的内容和变更

支持的对象类型包括：

**提交（commit）**：显示提交的作者、时间、提交信息，以及该提交的代码变更（diff）；

**标签（tag）**：显示标签的名称、创建者、备注，以及标签指向的提交；

**分支（branch）**：等同于查看该分支当前指向的提交；

**暂存区 / 工作区**：显示未提交的代码变更；

**树对象（tree）/blob 对象**：显示文件或目录的原始内容。

## 概述

```bash
# <Git对象> 可省略，默认显示 HEAD（当前提交）
# <Git对象>：可以是提交哈希、分支名、标签名、HEAD 别名（如 HEAD~1）、暂存区标识（如 :/README.md）等；
git show [<选项>] [<对象>…​]
```

## 与 `git log` 的区别

- `git log`: 显示提交历史列表，可以按时间、作者、文件等过滤。

- `git show`: 显示单个对象（提交、标签、树、文件等）的详细信息。

## 参数

### 基础参数（指定操作对象）

这类参数用于明确要查看的 Git 对象，可省略（默认查看 HEAD）。

#### `<Git对象>`: 查看的对象

- **作用**: 指定要查看的对象，支持提交哈希、分支名、标签名、HEAD 别名、文件路径等。
- **默认值**: `HEAD`
- **示例**: `git show a1b2c3d`

### 输出内容控制参数

这类参数决定显示「提交元信息」「代码变更」「文件列表」等哪类内容。

#### `--no-patch, -s`: 忽略代码变更（diff）

- **作用**: 只显示提交的元信息（作者、时间、提交信息），**完全忽略代码变更（diff）**。
- **示例**
  ```bash
  git show --no-patch a1b2c3d
  # 输出（无diff）：
  # commit a1b2c3d4e5f67890abcdef1234567890abcdef12
  # Author: 用户名 <user@example.com>
  # Date:   Wed Mar 4 10:00:00 2026 +0800
  #
  #     新增登录功能
  ```

#### `--patch, -p`: 显示忽略代码变更（diff）

- **作用**: 显示代码变更（diff）。默认行为， 一般无需使用此参数。
- **使用场景**: 在某些情况下，你可能需要强制显示补丁，尤其是当与其他选项组合使用时。例如与 `--stat` 组合

#### `--stat`: 显示变更文件的统计信息

- **作用**: 只显示变更文件的统计信息（文件名、增删行数），不显示具体 diff 内容。
- **输出格式**: `文件名 | 总行数 + 新增行数(+) - 删除行数(-)`
- **示例**
  ```bash
  git show --stat a1b2c3d
  # 输出：
  # commit a1b2c3d4e5f67890abcdef1234567890abcdef12
  # Author: 用户名 <user@example.com>
  # Date:   Wed Mar 4 10:00:00 2026 +0800
  #
  #     新增登录功能
  # 文件路径       | 3 +++++++++++++++++++--
  # 1 file changed, 2 insertions(+), 1 deletion(-)
  ```

#### `--name-only`: 显示本次提交变更的文件名列表

- **作用**: 只显示本次提交变更的文件名列表，无任何其他信息
- **适用场景**: 快速确认某提交修改了哪些文件

#### `--name-status`: 显示变更文件名 + 变更状态

- **作用**: 显示变更文件名 + 变更状态（状态码说明：A = 新增，M = 修改，D = 删除，R = 重命名，C = 复制）。
- **适用场景**: 不仅要知道改了哪些文件，还要知道文件的变更类型

#### `--raw`: 以 Git 原始格式输出

- **作用**: 以 Git 原始格式显示变更信息（包含 blob 对象哈希、文件模式、状态码），适合脚本解析。

### 格式控制参数（调整输出样式）

#### `--pretty=<格式>, --format=<格式>`: 指定提交元信息的输出格式

- **作用**: 指定提交元信息的输出格式，支持预设格式和自定义格式。
- **预设格式**:
  - `oneline`: 单行显示（哈希 + 提交信息）；
  - `oneline`：单行显示（哈希 + 提交信息）；
  - `short`：精简格式（作者 + 提交信息）；
  - `full`：完整格式（作者 + 提交者 + 提交信息）；
  - `fuller`：更完整格式（含作者 / 提交者的时间）；
  - `raw`：原始 Git 对象格式；
  - `format:<自定义>`：自定义格式（常用占位符：% H = 完整哈希，% h = 简写哈希，% an = 作者名，% ae = 作者邮箱，% ad = 作者时间，% s = 提交信息）。

#### `--abbrev-commit, --no-abbrev-commit`: 提交哈希输出格式

- **作用**: `--abbrev-commit` 简写提交哈希（默认前 7 位），`--no-abbrev-commit` 显示完整哈希。

#### `--date=<格式>`: 定制时间的显示格式

- **作用**: 定制时间的显示格式，配合 `--pretty` 使用。
- **取值**：`relative`（相对时间，如 “2 天前”）、`iso`（ISO 格式）、`local`（本地时间）、`raw`（时间戳）。

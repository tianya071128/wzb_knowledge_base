# 问题

## **user.name** 配置项的作用

此配置**对远程服务的身份验证没有任何影响**，只是用来作为提交的名称信息

## 已跟踪文件, 在 `.gitignore` 文件添加不会被忽略

`.gitignore` 文件是用来确保 Git **未跟踪的文件**保持不被跟踪, 已跟踪的文件还是保持跟踪状态。

此时通过命令 `git rm --cached <file/pattern>` 移除跟踪状态, 就会被 `.gitignore` 文件忽略

## 如何确定当前仓库的推送方式

使用命令 `git remote -v`:

- 如果是 **`HTTPS` 方式**

  ```plaintext
  origin  https://github.com/xxx/xxx.git (fetch)
  origin  https://github.com/xxx/xxx.git (push)
  ```

  - 以 `https://` 开头
  - 每次推送可能要输账号密码 /token

- 如果是 **SSH 方式**

  ```plaintext
  origin  git@github.com:xxx/xxx.git (fetch)
  origin  git@github.com:xxx/xxx.git (push)
  ```

  - 以 `git@` 开头
  - 不需要密码，用密钥认证

使用如下命令切换推送方式

```bash
# 切换成 HTTPS
git remote set-url origin https://github.com/用户名/仓库名.git

# 切换成 SSH
git remote set-url origin git@github.com:用户名/仓库名.git
```

## HTTP/S 如何切换凭证保存方式

`HTTP/S` 模式下需要输入账号密码生成凭证, 但是频繁输入账号密码会非常繁琐。此时可以配置 [`credential.helper`](/git/git-config.html#credential-helper) 来缓存其凭证, 无需重复输入账号密码

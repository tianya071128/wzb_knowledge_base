# 元注解

元注解是"注解的注解"，用于约束和定义自定义注解的行为。Java 提供了以下元注解：

| 元注解 | 作用 |
| --- | --- |
| `@Target` | 指定注解可以标注的程序元素 |
| `@Retention` | 指定注解的生命周期 |
| `@Documented` | 标记注解会出现在 Javadoc 文档中 |
| `@Inherited` | 标记注解可以被子类继承 |
| `@Repeatable` | 允许同一注解在同一目标上重复使用 |

## @Target

指定注解可以作用在哪些程序元素上，取值为 `ElementType` 枚举：

```java
@Target({ElementType.TYPE, ElementType.METHOD})
public @interface MyAnnotation {}
// 该注解只能用在类/接口和方法上
```

常用取值：

| 值 | 可标注的元素 |
| --- | --- |
| `TYPE` | 类、接口、枚举、注解 |
| `METHOD` | 方法 |
| `FIELD` | 字段 |
| `PARAMETER` | 方法参数 |
| `CONSTRUCTOR` | 构造方法 |
| `LOCAL_VARIABLE` | 局部变量 |
| `ANNOTATION_TYPE` | 注解类型 |
| `PACKAGE` | 包 |

## @Retention

指定注解的生命周期，取值为 `RetentionPolicy` 枚举：

```java
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {}
// 该注解在运行时仍可通过反射获取
```

| 值 | 说明 |
| --- | --- |
| `SOURCE` | 仅存在于源码，编译后丢弃（如 `@Override`） |
| `CLASS` | 保留到 class 文件，运行时不可见（默认值） |
| `RUNTIME` | 运行时可见，可通过反射读取（如 Spring 注解） |

选择依据：
- 仅给编译器用的 → `SOURCE`
- 编译时处理工具用的 → `CLASS`
- 需要运行时反射读取的 → `RUNTIME`

## @Documented

标记注解会被 Javadoc 工具记录到 API 文档中：

```java
@Documented
@Retention(RetentionPolicy.RUNTIME)
public @interface ApiDoc {}
```

默认情况下，注解不会出现在 Javadoc 生成的文档中，加上 `@Documented` 后，使用该注解的类/方法会在文档中显示注解信息。

## @Inherited

标记注解可以被子类继承（仅对类上的注解生效）：

```java
@Inherited
@Retention(RetentionPolicy.RUNTIME)
public @interface BaseService {}

@BaseService
public class Parent {}

// Child 自动继承 @BaseService
public class Child extends Parent {}
```

注意：
- 只对类生效，方法和字段上的注解不会被继承
- 子类如果自身标注了同类型注解，会覆盖父类的

## @Repeatable

允许同一注解在同一目标上重复使用（Java 8+ 引入）：

```java
// 1. 定义容器注解
public @interface Schedules {
    Schedule[] value();
}

// 2. 标记可重复
@Repeatable(Schedules.class)
public @interface Schedule {
    String cron();
}

// 3. 使用
@Schedule(cron = "0 0 8 * * ?")
@Schedule(cron = "0 0 20 * * ?")
public void syncData() {}
```

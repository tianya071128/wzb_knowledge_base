# 内置注解

JDK 内置注解是由 Java 标准库提供的注解，无需引入额外依赖。

## 编译器相关

### @Override

标记方法重写了父类/接口的方法。若方法签名不匹配，编译器报错：

```java
public class Animal {
    public void speak() {}
}

public class Dog extends Animal {
    @Override
    public void speak() { // 正确重写
        System.out.println("Woof");
    }
}
```

写错方法名时编译器会提示：

```java
@Override
public void speakk() { // 编译错误：method does not override
    System.out.println("Woof");
}
```

### @Deprecated

标记元素已过时，使用时编译器产生警告，IDE 会显示删除线：

```java
@Deprecated
public class OldService {
    @Deprecated
    public void oldMethod() {}
}

// 调用时会有警告
new OldService().oldMethod(); // ⚠️ deprecated
```

可以配合 `@Deprecated(since, forRemoval)` 使用（Java 9+）：

```java
@Deprecated(since = "2.0", forRemoval = true)
public void legacyMethod() {}
```

### @SuppressWarnings

抑制编译器指定的警告信息：

```java
@SuppressWarnings("unchecked")
public void process(List rawList) {
    List<String> list = rawList; // 不会产生 unchecked 警告
}
```

常用警告类型：

| 值 | 说明 |
| --- | --- |
| `"unchecked"` | 未检查的类型转换警告 |
| `"deprecation"` | 使用已过时元素的警告 |
| `"rawtypes"` | 使用原始类型（无泛型参数）的警告 |
| `"unused"` | 未使用的变量/方法警告 |
| `"serial"` | 可序列化类缺少 serialVersionUID 的警告 |

可抑制多个警告：

```java
@SuppressWarnings({"unchecked", "deprecation"})
public void doSomething() {}
```

### @SafeVarargs

抑制泛型可变参数的堆污染警告，用于 `final`、`static` 或 `private` 方法（Java 7+）：

```java
@SafeVarargs
public final void printAll(List<String>... lists) {
    for (List<String> list : lists) {
        list.forEach(System.out::println);
    }
}
```

### @FunctionalInterface

标记接口为函数式接口（只有一个抽象方法），若不符合则编译报错：

```java
@FunctionalInterface
public interface Comparator<T> {
    int compare(T o1, T o2);
}

// 若添加第二个抽象方法，编译器报错
```
